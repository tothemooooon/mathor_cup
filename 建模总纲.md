# MathorCup 2026 A题建模总纲（项目记忆）

## 0. 项目纲领（记忆锚点）

### 0.1 核心目标
- 目标不是“做出一个能跑的模型”，而是“在竞赛评分规则下最大化总分”。
- 每一个建模与工程决策都要回答三个问题：
  1. 是否提升可解释性（评委能快速看懂并认可）
  2. 是否体现方法专业性（数学建模深度与量子/QUBO契合）
  3. 是否保持代码简洁可复现（提交前稳定复跑）

### 0.2 评分导向权重（固定）
- 可解释性 40%
- 方法专业性 40%
- 代码简洁性 20%

### 0.3 统一决策原则
- 原则 A：先可行，再最优。不可行方案不进入主线。
- 原则 B：先 baseline，再创新。每一问先建立可复现 baseline，再做改进。
- 原则 C：改动必须可追溯。每次改动必须写明“原因-动作-结果-去留”。
- 原则 D：以统一结果契约记录实验，避免“口头最优”。

## 2. 评委视角题目解读（提分导向）

### 2.1 评委关心的不是“是否复杂”，而是“是否完整”
- A题本质是 VRP 的分层难化：
  - Q1：单车 TSP（无时间窗、无容量）
  - Q2：单车 + 时间窗惩罚
  - Q3：大规模单车（可解性挑战）
  - Q4：多车 + 容量 + 时间窗 + 车辆数权衡
- 评委期待看到“从小到大、从简到繁”的连续技术路线，而非四问割裂。

### 2.2 可以超越多数选手的关键点
- 关键点 1：统一建模母体
  - 说明四问如何共享变量定义、目标拆分和惩罚逻辑。
- 关键点 2：规模可解性论证
  - Q3 不能只给结果，必须解释为何需要分解、分解损失是什么、如何补偿。
- 关键点 3：权重与参数的可解释调参
  - 展示惩罚系数、车辆权重等参数对结果的影响，不做“黑箱拍脑袋”。
- 关键点 4：真机叙事闭环
  - 即使主求解在模拟退火，也要规划真机尝试记录与哈密顿量曲线展示路径。

### 2.3 常见失分点预警
- 只给最优路径，不给约束校验。
- 只给最终参数，不解释参数选择过程。
- Q3/Q4 只堆算法，不解释复杂度与可复现性。
- 文档与代码不一致，结果不可复跑。

## 4. Q1-Q4 建模路线（导师视角指导）

### 4.1 Q1（15客户，单车，无时间窗无容量）
- 问题类型：标准 TSP 的 QUBO 表达。
- 变量：`x[i,t]` 表示客户 `i` 是否处于访问序位 `t`。
- 核心难点：
  - 惩罚项过小导致不可行；过大导致目标函数梯度被淹没。
  - 退火随机性导致结果波动。
- 处理方法：
  - 双约束平方惩罚（每位唯一 + 每客唯一）
  - 多 seed 取优 + 固定 seed 复现
- 选择原因：
  - 形式标准、解释清晰、与后续问题可复用。

### 4.2 Q2（15客户，单车，时间窗惩罚）
- 问题类型：TSP + 时间相关惩罚（软约束）。
- 关键思想：将时间窗惩罚直接写入 QUBO 一次项（B），并用迭代修正到达时间估计（C）。
- 难点：
  - 到达时刻与路径强耦合，直接精确入模会引入大量变量。
  - 惩罚估计过粗时，可能导致权重敏感性不足。
- 处理方法：
  - 先估计位置级到达时间 `tau_k`，构造 `penalty(j,k)`。
  - 将 `tw_weight * penalty(j,k) * x[j,k]` 加入目标函数一次项（Q 对角）。
  - 迭代 3-5 轮：用上一轮真实到达时间更新 `tau_k`，重建 QUBO 再求解。
- 选择原因：
  - 变量规模保持 225，不增加二进制变量，工程上可落地且论文叙事完整。

### 4.3 Q3（50客户，单车大规模）
- 问题类型：大规模 QUBO 可解性瓶颈。
- 关键思想：分层混合求解。
  - 先分解子问题（聚类/分片）
  - 子问题内 QUBO 求解
  - 簇间拼接 + 局部改进
- 难点：
  - 分解会损失全局最优性。
- 处理方法：
  - 明确分解准则 + 拼接优化 + 消融验证“分解是否值得”。
- 选择原因：
  - 兼顾“可算性”和“量子求解叙事”。

### 4.4 Q4（50客户，多车+容量+时间窗）
- 问题类型：多目标 VRP（车数 + 路程 + 时间窗违反）。
- 关键思想：两阶段求解。
  - 阶段1：客户分配到车辆（容量可行优先）
  - 阶段2：车内路径优化（QUBO）
- 难点：
  - 车辆数最少与总运输时间最短之间矛盾明显。
- 处理方法：
  - 车辆数扫描，绘制目标变化曲线，给出权重驱动的管理建议。
- 选择原因：
  - 工程上可控，便于解释“决策层含义”。
